## 📋 Шаги для запуска

### 1. Установка PostgreSQL
- [ ] Установить PostgreSQL (если еще не установлен)
- [ ] Создать базу данных `clienthub`

### 2. Настройка Backend
```bash
cd clienthub-backend
npm install
copy .env.example .env
# Отредактировать .env (указать пароль PostgreSQL)
npm run build
npm run db:migrate
npm run db:seed  # Опционально: добавить демо-данные
npm run dev
```

### 3. Настройка Frontend
```bash
cd clienthub-frontend
npm install
npm run dev
```

### 4. Тестирование
- [ ] Открыть http://localhost:5173
- [ ] Зарегистрироваться или войти (demo@clienthub.com / demo123)
- [ ] Создать клиента
- [ ] Открыть карточку клиента
- [ ] Добавить заказ
- [ ] Добавить взаимодействие
- [ ] Добавить комментарий

## 🎯 Основные сценарии для демонстрации

### Сценарий 1: Добавление нового клиента
1. Войти в систему
2. Нажать "+ Добавить клиента"
3. Заполнить форму (имя, телефон, email, компания, теги, заметки)
4. Сохранить
5. Клиент появляется в списке

### Сценарий 2: Работа с историей клиента
1. Открыть карточку клиента
2. Перейти на вкладку "Заказы"
3. Добавить заказ с суммой и статусом
4. Перейти на вкладку "Взаимодействия"
5. Добавить взаимодействие (звонок, встреча, email)
6. Перейти на вкладку "Комментарии"
7. Добавить комментарий

### Сценарий 3: Отслеживание повторных касаний
1. Просмотреть историю взаимодействий клиента
2. Увидеть все точки контакта (звонки, встречи, сообщения)
3. Просмотреть комментарии с датами
4. Увидеть полную картину работы с клиентом

## 🐛 Решение проблем

### Ошибка подключения к БД:
- Проверить, что PostgreSQL запущен
- Проверить DATABASE_URL в .env
- Проверить, что база данных создана

### Порт занят:
- Изменить PORT в .env (backend)
- Изменить порт в vite.config.ts (frontend)

### CORS ошибки:
- Убедиться, что backend запущен
- Проверить URL в src/api/axios.ts
