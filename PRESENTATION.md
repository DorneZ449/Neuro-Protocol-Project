# ClientHub - Презентация проекта

## Слайд 1: Титульный
**ClientHub**
CRM-система для учёта клиентов и истории взаимодействия

Разработано с использованием ИИ (искусственного интеллекта)

---

## Слайд 2: Проблема

### Что теряет малый бизнес?

❌ **Контакты разбросаны** - телефон, блокноты, Excel, мессенджеры
❌ **Потеря истории** - забывают, что заказывал клиент, когда общались
❌ **Упущенные продажи** - не отслеживают повторные касания
❌ **Хаос в работе** - нет единой системы, каждый работает по-своему

### Результат:
- Клиенты уходят к конкурентам
- Теряются деньги
- Снижается качество обслуживания

---

## Слайд 3: Решение - ClientHub

### Простая CRM для малого бизнеса

✅ **Единая база клиентов** - все контакты в одном месте
✅ **История заказов** - видно, что и когда покупал клиент
✅ **История взаимодействий** - все звонки, встречи, сообщения
✅ **Комментарии и заметки** - важная информация не теряется
✅ **Отслеживание повторных касаний** - не упускаем клиентов

---

## Слайд 5: Основной сценарий использования

### Путь менеджера:

1. **Добавляет клиента** → имя, телефон, email, компания, теги
2. **Создаёт заказ** → название, сумма, статус, дата
3. **Фиксирует взаимодействие** → звонок, встреча, email, сообщение
4. **Оставляет комментарий** → важные заметки о клиенте
5. **Видит полную историю** → все касания с клиентом в одном месте

### Результат:
Менеджер всегда знает, на каком этапе клиент и что с ним делать дальше

## Слайд 6: Архитектура приложения

**Frontend:**
- React 18 + TypeScript
- Tailwind CSS
- React Router
- React Query

**Backend:**
- Node.js + Express
- TypeScript
- PostgreSQL
- JWT аутентификация

**Архитектура:** REST API, разделение frontend/backend

## Слайд 7: Структура базы данных

### 5 основных таблиц:

1. **users** - пользователи системы (сотрудники)
2. **clients** - база клиентов
3. **orders** - заказы клиентов
4. **interactions** - история взаимодействий
5. **comments** - комментарии к клиентам

**Связи:** Foreign keys, cascade delete, правильная нормализация
